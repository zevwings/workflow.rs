# 📝 样式规范指南

> 统一检查指南文档的术语、格式和视觉风格，提升一致性和可读性。

## 🎯 术语规范

### 检查类型术语

| 术语 | 含义 | 使用场景 |
|------|------|----------|
| **快速检查** | 5-15分钟的日常检查 | 提交前验证 |
| **深入检查** | 2-4小时的全面检查 | 功能完成后、定期审查 |
| **专门检查** | 针对特定领域的检查 | CLI、代码、测试、文档 |
| **跨领域分析** | 识别不同领域间的问题关联 | 深入检查的高级分析 |

### 优先级术语

| 优先级 | 符号 | 含义 | 处理方式 |
|--------|------|------|----------|
| **P0** | 🚨 | 必须修复 | 阻止提交，立即修复 |
| **P1** | ⚠️ | 建议修复 | 本次或下次提交修复 |
| **P2** | 💡 | 可选修复 | 记录到技术债务，择机修复 |

### 状态术语

| 状态 | 符号 | 含义 |
|------|------|------|
| **通过** | ✅ | 检查完全通过 |
| **部分通过** | ⚠️ | 有问题但不阻止进行 |
| **未通过** | ❌ | 检查失败，需要修复 |
| **进行中** | 🔄 | 正在检查或修复 |
| **已完成** | ✅ | 任务已完成 |
| **待处理** | 📋 | 等待处理 |

### 标题层级规范

```markdown
# 一级标题：文档标题
## 二级标题：主要章节
### 三级标题：子章节
#### 四级标题：具体检查项
##### 五级标题：详细说明（尽量避免使用）
```

## 📋 检查清单格式

### 标准检查清单

```markdown

- [ ] 检查项1：具体描述
- [ ] 检查项2：具体描述
- [ ] 检查项3：具体描述
```

### 分类检查清单

```markdown
#### 必须完成（P0）
- [ ] 关键检查项1
- [ ] 关键检查项2

#### 建议完成（P1）
- [ ] 重要检查项1
- [ ] 重要检查项2

#### 可选完成（P2）
- [ ] 可选检查项1
- [ ] 可选检查项2
```
